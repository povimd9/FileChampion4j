File\+Champion4j is a powerful and flexible Java library for validating and processing files. The library can be used to check files for a variety of properties, including mime type, magic bytes, header signatures, footer signatures, maximum size, and more. The library can also execute extension plugins that are defined for the file type.

{\bfseries{See \href{https://github.com/povimd9/FileChampion4j/wiki}{\texttt{ File\+Champion4j Wiki}} for detailed instructions on configurations and usage.}}\hypertarget{md__c___users__user1_git__file_champion4j__r_e_a_d_m_e_autotoc_md14}{}\doxysubsection{Features}\label{md__c___users__user1_git__file_champion4j__r_e_a_d_m_e_autotoc_md14}

\begin{DoxyItemize}
\item Easy to understand and configure for developers, operations, and security engineers.
\item JSON-\/based configuration, supporting the ability to separate configurations from code.
\item Flexible to support various integrations, including client-\/defined controls.
\item Support for in-\/memory and on-\/disk validation.
\item Validate files for a variety of properties, including mime type, magic bytes, header signatures, footer signatures, maximum size, filename cleanup/encoding, and owner/permissions of file.
\item Custom plugins execution support for extended usability.
\item Comprehensive error handling and reporting.

The File\+Validator class is initialized with a JSON configuration object. The File\+Validator.\+validate\+File() method is the main entry point for validating files. validate\+File() takes 4 arguments\+:


\begin{DoxyItemize}
\item Configuration category -\/ the category defined under the \char`\"{}\+Validations\char`\"{} object (e.\+g. \char`\"{}\+Documents\char`\"{})
\item Target file bytes -\/ a byte array of file content to be scanned
\item File name -\/ the original file name associated with the target file bytes
\item Output directory -\/ (optional) directory to which a validated file will be saved to (required for owner/permissions change)
\end{DoxyItemize}

validate\+File() returns a Validation\+Response object, containing\+:


\begin{DoxyItemize}
\item is\+Valid -\/ a boolean indicating whether the file is valid or not
\item results\+Info -\/ a string containing additional information about the validation pass/fail status
\item file\+Bytes -\/ the file bytes if the file is valid (original of new file if was modified by plugins)
\item file\+Checksum\+: the SHA-\/256 checksum of the file
\end{DoxyItemize}\hypertarget{md__c___users__user1_git__file_champion4j__r_e_a_d_m_e_autotoc_md21}{}\doxysubsection{Configuration}\label{md__c___users__user1_git__file_champion4j__r_e_a_d_m_e_autotoc_md21}
Example configuration of validation controls and plugins. \char`\"{}\+Validations\char`\"{} and at least 1 child key is required for validation to work. Plugins and steps can be defined as necessary -\/ current implementation only supports process execution, API support will come soon.


\begin{DoxyItemize}
\item \$\{file\+Path\}, and \$\{file\+Checksum\} -\/ can be used in \textquotesingle{}endpoint\textquotesingle{} value to inject correspanding values
\item \$\{file\+Content\} -\/ can be used in \textquotesingle{}endpoint\textquotesingle{} value to inject file content in base64 encoding
\item \$\{STEP\+\_\+\+NAME.\+file\+Path\} -\/ can be used to extract file path content and replace original file bytes
\item \$\{STEP\+\_\+\+NAME.\+file\+Content\} -\/ can be used to extract base64 file content from base64 encoded bytes
\end{DoxyItemize}
